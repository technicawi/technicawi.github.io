Slither — Smart Contract Static Analyzer

What is Slither?

Slither is a static analysis framework for Solidity smart contracts created by Trail of Bits.

It analyzes Solidity source code without executing it and detects:

  • Reentrancy
  • Access control flaws
  • Unchecked return values
  • Dangerous delegatecalls
  • Storage collisions
  • Upgradeable proxy issues
  • Uninitialized state variables
  • Incorrect inheritance usage

Install Methods

1. Tool Installation (Recommended)

curl -LsSf https://astral.sh/uv/install.sh | sh

uv tool install slither-analyzer

Installs Slither in an isolated environment and exposes the slither command globally.

Best for auditors, bug bounty hunters, and offensive security workflows.

2. Full Python Environment Installation

python -m venv venv
source venv/bin/activate

pip install slither-analyzer

Required if writing custom plugins, detectors, or importing Slither into Python scripts.

Basic Usage

slither .

slither contracts/Token.sol

Slither automatically compiles contracts using:

  • Foundry
  • Hardhat
  • Truffle
  • solc

Common Commands

List Detectors

slither --list-detectors

Generate Call Graph

slither . --print call-graph

Generate CFG (Control Flow Graph)

slither . --print cfg

Inheritance Graph

slither . --print inheritance-graph

Function Summary

slither . --print function-summary

Example Vulnerable Contract

contract Vault {

    mapping(address => uint) public balances;

    function withdraw() public {
        uint amount = balances[msg.sender];

        (bool ok,) = msg.sender.call{value: amount}("");

        require(ok);

        balances[msg.sender] = 0;
    }
}

Slither detects: 

Reentrancy in Vault.withdraw()

Operational Notes

  • Static analysis does not execute code.
  • False positives exist.
  • Always manually verify findings.
  • Combine with dynamic testing and invariant fuzzing.
  • Works extremely well with Foundry.
Slither is reconnaissance, not proof of exploitability.

Useful Workflow

# Compile contracts
forge build

# Run slither
slither .

# Export findings
slither . --json results.json

# Generate graphs
slither . --print call-graph
